# TestApp v2 — Платформа онлайн-тестирования

Полнофункциональное веб-приложение для создания и прохождения тестов с разграничением прав доступа по ролям. Поддерживает вопросы с одиночным и множественным выбором, теги, историю результатов и административную панель.

---

## Стек технологий

| Слой | Технологии |
|------|------------|
| Backend | Python 3.10+, FastAPI, SQLAlchemy (ORM) |
| База данных | PostgreSQL |
| Frontend | React 18 + Vite |
| Аутентификация | JWT-токены (python-jose), bcrypt |
| Валидация | Pydantic v2 |

---

## Роли пользователей

| Роль | Возможности |
|------|-------------|
| **Гость** | Просмотр списка тестов, прохождение тестов без сохранения результата |
| **Клиент** | Всё что доступно гостю + сохранение результатов, история прохождений, личная статистика |
| **Администратор** | Всё что доступно клиенту + создание/редактирование/удаление тестов, управление пользователями и их ролями, просмотр всех результатов |

## Установка и запуск

### Требования

- Python 3.10+
- Node.js 18+
- PostgreSQL 14+

---

### 1. Создать базу данных

Откройте pgAdmin 4 или psql и выполните:

```sql
CREATE DATABASE testdb02;
```

---

### 2. Настроить подключение к БД

Откройте `backend/database.py` и укажите ваши данные:

```python
DATABASE_URL = "postgresql://postgres:ВАШ_ПАРОЛЬ@localhost:5432/testdb02"
```

---

### 3. Запуск бэкенда

```bash
cd backend

# Создать виртуальное окружение
python -m venv .venv

# Активировать (Windows)
.venv\Scripts\activate

# Активировать (Linux / macOS)
source .venv/bin/activate

# Установить зависимости
pip install -r requirements.txt

# Запустить сервер
uvicorn main:app --reload
```

- Бэкенд доступен по адресу: `http://localhost:8000`
- Интерактивная документация API (Swagger): `http://localhost:8000/docs`
- Альтернативная документация (ReDoc): `http://localhost:8000/redoc`

> Таблицы создаются автоматически при первом запуске через `Base.metadata.create_all`.

---

### 4. Запуск фронтенда

```bash
cd frontend

# Установить зависимости
npm install

# Запустить dev-сервер
npm run dev
```

- Фронтенд доступен по адресу: `http://localhost:5173`

## Создание первого администратора

1. Зарегистрируйтесь через сайт (страница «Регистрация»).
2. Откройте pgAdmin 4 или psql и выполните:

```sql
\c testdb02
UPDATE users SET role = 'admin' WHERE username = 'ваш_логин';
```

3. Выйдите и войдите повторно — в навигации появится кнопка «⚙️ Админ».

## Схема базы данных

```
users
  id, username (unique), password (bcrypt), role (client | admin)

tests
  id, title

tags
  id, name (unique)

test_tags  (many-to-many: tests ↔ tags)
  test_id, tag_id

questions
  id, test_id → tests.id, text, question_type (single | multi)

question_options
  id, question_id → questions.id, text, is_correct, order

results
  id, user_id → users.id, test_id → tests.id (nullable),
  test_title_snapshot, total_questions_snapshot, score
```

### Важные детали схемы

- `question_type` — тип вопроса: `single` (один правильный ответ) или `multi` (несколько правильных ответов).
- `question_options` — варианты ответов вынесены в отдельную таблицу (2–30 вариантов на вопрос).
- `test_title_snapshot` и `total_questions_snapshot` в `results` — снапшоты данных теста на момент прохождения. Позволяют сохранить историю даже если тест впоследствии будет удалён.
- Каскадное удаление: при удалении теста автоматически удаляются все его вопросы, варианты ответов и связи с тегами.

## API — справочник эндпоинтов

### Публичные (без авторизации)

| Метод | URL | Описание |
|-------|-----|----------|
| `POST` | `/register` | Регистрация нового пользователя |
| `POST` | `/login` | Вход, возвращает JWT-токен |
| `GET` | `/tests` | Список тестов (параметры: `search=`, `tags=`, `page=`) |
| `GET` | `/tests/{id}` | Детали теста с вопросами и вариантами ответов |
| `GET` | `/tags` | Список всех тегов |

### Для авторизованных клиентов

| Метод | URL | Описание |
|-------|-----|----------|
| `GET` | `/me` | Информация о текущем пользователе |
| `POST` | `/submit` | Отправить ответы на тест, получить результат |
| `GET` | `/my-results` | История всех результатов пользователя |
| `GET` | `/my-stats` | Общая статистика (кол-во тестов, средний балл и т.д.) |

### Только для администраторов

| Метод | URL | Описание |
|-------|-----|----------|
| `GET` | `/admin/tests-list` | Список тестов с поиском и пагинацией |
| `GET` | `/admin/tests/{id}` | Получить тест для редактирования |
| `POST` | `/admin/tests` | Создать новый тест |
| `PUT` | `/admin/tests/{id}` | Обновить существующий тест |
| `DELETE` | `/admin/tests/{id}` | Удалить тест (каскадно) |
| `DELETE` | `/admin/tests-all` | Удалить все тесты |
| `GET` | `/admin/tests-export` | Экспорт всех тестов в JSON |
| `POST` | `/admin/tests-import` | Импорт тестов из JSON (`?skip_duplicates=true`) |
| `GET` | `/admin/users` | Список всех пользователей |
| `PUT` | `/admin/users/{id}/role` | Изменить роль пользователя |
| `DELETE` | `/admin/users/{id}` | Удалить пользователя |
| `GET` | `/admin/results` | Все результаты всех пользователей |
| `DELETE` | `/admin/results/{id}` | Удалить конкретный результат |

## Правила валидации

| Поле | Ограничение |
|------|-------------|
| Имя пользователя | 3–40 символов, без пробелов по краям |
| Пароль | 6–40 символов |
| Название теста | 1–100 символов |
| Текст вопроса | 1–200 символов |
| Вариант ответа | 1–200 символов |
| Вариантов на вопрос | минимум 2, максимум 30 |
| Вопросов в тесте | минимум 1, максимум 100 |
| Тег | до 40 символов, приводится к нижнему регистру |
| Правильных ответов | минимум 1 на каждый вопрос |

## Аутентификация

- Используются JWT Bearer-токены.
- Срок жизни токена: **60 минут**.
- Алгоритм подписи: **HS256**.
- Пароли хранятся в виде bcrypt-хэша.
- Секретный ключ задаётся в `auth.py` (переменная `SECRET_KEY`) — замените на надёжный ключ в продакшене.

## Полезные SQL-команды

```sql
-- Подключиться к базе
\c testdb02

-- Назначить администратора
UPDATE users SET role = 'admin' WHERE username = 'логин';

-- Снять права администратора
UPDATE users SET role = 'client' WHERE username = 'логин';

-- Посмотреть всех пользователей
SELECT id, username, role FROM users;

-- Очистить ВСЕ данные (порядок важен из-за внешних ключей)
TRUNCATE question_options, questions, test_tags, tests, tags, results, users RESTART IDENTITY CASCADE;

-- Очистить только тесты и всё связанное (вопросы, теги, результаты)
TRUNCATE question_options, questions, test_tags, tests, tags RESTART IDENTITY CASCADE;

-- Удалить только тесты (вопросы удалятся каскадом, результаты останутся)
DELETE FROM tests;

-- Очистить только результаты
TRUNCATE results RESTART IDENTITY;
```

## Изменения в v2 (относительно v1)

- **Новое поле `question_type`** в таблице `questions` — поддержка вопросов с множественным выбором (`multi`).
- **Новая таблица `question_options`** — варианты ответов вынесены из JSON-поля в отдельную таблицу с флагом `is_correct` и порядком отображения `order`.
- **Новые таблицы `tags` и `test_tags`** — теги для тестов с поддержкой отношения many-to-many.
- **Новые поля в `results`** — `test_title_snapshot` и `total_questions_snapshot` сохраняют название теста и количество вопросов в момент прохождения (защита от потери истории при удалении теста).
